# gradekeeper

A grade computation and reporting pipeline for university courses. The core
engine is an [Idris 2](https://idris-lang.org) program that reads a course
specification and student score data, computes final grades, and produces a
Markdown report. Supporting [Guile Scheme](https://www.gnu.org/software/guile/)
scripts and a `Makefile` handle data wrangling and glue the pipeline together.

## Overview

The pipeline takes three inputs for a given course:

1. A **course specification** (`.scm` s-expression file) describing score
   items, grade formulas, grade cutoffs, and any per-student exceptions.
2. A **Canvas grade export** (CSV) downloaded from the Canvas LMS.
3. A **SIS enrollment export** (`.xlsx`) from the university's student
   information system.

It produces:

- A Markdown grade report (`<course>--<term>--grade-report.md`)
- HTML and PDF renderings of that report (via [pandoc](https://pandoc.org))

## Course directories

Each course directory should contain:

- a symlink `Makefile` pointing to the makefile `per-course-makefile.lk` found in 
  the `course-specs` directory of the repo. 

  I keep this file in the super-directory of my course directories,
  like this:
  
  ```
     ├── AY2025-2026--2025-fa--Math245-data
     ├── AY2025-2026--2026-sp--Math087-data
     ├── AY2025-2026--2026-sp--Math136-data
     ├── Makefile.lk
  
  ```
- an `xls` file obtained from `SIS` describing the course enrollment

- a `csv` file obtained form `Canvas` containing the grades for the course.  
  Note that the filename should contain the word `Grades`.
  
- a `scm` file course specification file.

Examples of these files can be found in the `example` directory.  If
you clone the repository, install `gradekeeper` and the scheme scripts
(via `make install`) you should be able to run `make` in the `example`
sub-directory.  (see `Dependencies` below).

## Running the pipeline

In a course data directory, run:

\```bash
make            # build everything: convert → scores → html → pdf
make scores     # just the Markdown grade report
make html       # Markdown → HTML (requires scores)
make convert    # just convert the spec .scm → generated JSON
\```

The pipeline converts the enrollment XLS to CSV, the spec SCM to JSON,
merges Canvas and enrollment data into per-student scores, runs the
`gradekeeper` binary to compute grades, and renders the report to HTML
and PDF via pandoc. Intermediate files are written to `generated/`.

## Course directory structure

See the `example` directory in the repo; it has the form:

```
├── course.mk
├── generated
│   ├── Math001-enrollment.csv
│   ├── Math001-spec.json
│   └── score-data.json
├── grade-exceptions.json
├── Makefile -> ../course-specs/per-course-makefile.lk
├── Math001-2023-2024-Fall--grade-report.html
├── Math001-2023-2024-Fall--grade-report.md
├── Math001-2023-2024-Fall--grade-report.pdf
├── Math001-canvas-Grades.csv
├── Math001-enrollment.xls
└── Math001-spec.scm
```


Notice that `course.mk` is generated by `course-spec-to-json.scm` --
don't hand-edit it.

## Dependencies

- **Idris 2** — the `gradekeeper` binary must be on your `PATH` (or set `GK`
  in the Makefile)
- **Guile Scheme** — for `course-spec-to-json.scm` and `grade-data.scm`
- **ssconvert** (from [gnumeric](https://wiki.gnome.org/Apps/Gnumeric)) — converts
  the SIS `.xlsx` enrollment file to CSV
- **pandoc** — for HTML and PDF output
- **unix2dos** — normalizes line endings in SIS/Canvas CSV exports

On a Guix system these can all be installed via `guix install` or declared in
your `guix home` configuration.

## Course specification format

Course specs are written as Guile s-expressions and converted to JSON by
`course-spec-to-json.scm` before being consumed by the Idris binary.

The format is illustrated by the [example](example/math001-spec.scm).



## Notes

- The Idris binary's JSON interface uses `ObjectWithSingleField` encoding for
  sum types (e.g. `{"Value": "hw1"}`). The `course-spec-to-json.scm` script
  is written to produce exactly this shape.
- Canvas CSV exports have timestamped filenames; the Makefile glob picks the
  most recent one automatically, so you can re-export from Canvas without
  renaming.
- SIS enrollment files often use Windows line endings and a "Class Roster"
  header row before the actual column headers; `grade-data.scm` handles both.
- Students marked as withdrawn in the SIS data are excluded from the report.
